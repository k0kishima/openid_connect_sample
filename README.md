# 開発環境構築

## この章の目的

OAuth2.0の登場人物各々に関して開発環境を用意する

## OAuth2.0 の登場人物

| ロール | 呼称 |
| ----- | --- |
| 発行されたアクセストークンを使ってリソースサーバーにアクセスするアプリケーション | クライアント |
| 正しいアクセストークンを使用しないとリソースが操作されないように保護を行っているサーバー | リソースサーバー |
| 委任認可の証としてアクセストークンを発行するサーバー | 認可サーバー |
| リソースの所有者であり、クライアント（後述）のエンドユーザー | リソースオーナー |

リソースオーナーが操作する端末（狭義ではブラウザ、広義ではそのプラットフォームとなっているマシンも含む）を「ユーザーエージェント」と呼ぶこともある

## サーバー一覧

yarn workspace を利用して下表の通り登場人物に対応するパッケージをそれぞれ用意した。

| 登場人物 | workspace でのパッケージ名 | URL |
| ------- | ---------------------- | --- |
| クライアント | client | http://localhost:3000/ |
| リソースサーバー | resource-server | http://localhost:3001/ |
| 認可サーバー | authorization-server | http://localhost:3002/ |

`yarn install` 実行後、 `yarn run wsrun -c start` で上記のサーバーを一括で起動することができる。
